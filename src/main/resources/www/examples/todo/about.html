This example allows you to interact with a list of todos. You can retrieve the list, add a new todo, or delete existing todos using dynamic routes and static/partial files.

This Todo app is server side rendered. Only HTMX and a handful of small JS scripts are used.

Some examples of the routes used:

  • GET /todos fetches the list of todos of the login user.
  • POST /todos creates a new todo, if logged in.
  • DELETE /todos deletes all todos, if logged in as ADMIN.

If you want to retrieve or manipulate a specific todo, you would use a dynamic route like /todos/:id. For example:

  • GET /todos/25 retrieves the todo with ID 25.
  • DELETE /todos/25 deletes the todo with ID 25.

The :id part of the URL is a dynamic parameter, and its value will be automatically extracted and passed to the handler method. You can access this parameter inside the method via the pathParams in the Yggdrasill.RequestHandler instance, like this:

String todoId = requestHandler.getPathParams().get("id");

There's @AccessControl annotation that you can use to restrict access to certain routes based on the JWT token.

To render this page, a layout system was made with partials, allow for dynamically show certain functions.

For example, only when a successful login is made, the form to create a new TODO is shown.
