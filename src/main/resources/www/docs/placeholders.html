Template Variables & Placeholders

JAWS now uses Thymeleaf as its template engine, providing robust and powerful templating capabilities. You can use Thymeleaf expressions to display dynamic information and include partial templates.

Modern Thymeleaf Syntax

  • th:insert="~{docs/header.html}": Insert a partial HTML template.
  • th:text="${variable}": Display text content safely (HTML escaped).
  • th:utext="${rawHtml}": Display raw HTML content (unescaped).
  • th:href="@{/path/to/page}": Generate URLs with context path.
  • th:if="${condition}": Conditional rendering.
  • th:each="item : ${list}": Loop through collections.

Available Utility Objects

  • ${utils.renderPartial('path/to/partial.html')}: Render a partial HTML file.
  • ${utils.getPathFor('file.txt')}: Get the full URL path for a file.
  • ${utils.getConfig('url')}: Get application configuration values.

Legacy Syntax (Still Supported)

For backward compatibility, the old syntax is automatically converted:

  • {{renderPartial("partial/header.html")}} → th:insert
  • {{variable}} → th:text="${variable}"

Setting Variables in Controllers

You can pass data to templates from your controllers using the Hermod utility methods:

// Set template variables
Hermod.setTemplateVariable("title", "Welcome to JAWS");
Hermod.setTemplateVariable("user", userObject);

// In template, access like:
// <h1 th:text="${title}"></h1>
// <span th:text="${user.name}"></span>

Request Parameters

Query and body parameters are automatically available in templates:

// Access individual parameters
<span th:text="${paramName}"></span>

// Or access parameter maps
<span th:text="${queryParams.paramName}"></span>
<span th:text="${bodyParams.paramName}"></span>
