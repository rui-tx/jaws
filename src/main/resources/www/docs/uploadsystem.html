Upload System

JAWS includes a comprehensive file upload system that allows users to upload, manage, and share files with custom expiry times. The system includes user authentication, file management, and automatic cleanup of expired files.

1. Basic Features

  • File upload with custom expiry times
  • User authentication and management
  • File management (list, download, delete)
  • Automatic cleanup of expired files
  • Drag and drop support
  • Progress tracking
  • File size limits (10MB)

2. Configuration

The upload system can be configured using the following environment variables:

  • UPLOADDIR: Specifies the directory where uploaded files will be stored. Defaults to examples/upload/files.

3. Database Schema

The upload system uses two main tables:

  • UPLOADS: Stores information about uploaded files
  • USER: Stores user authentication information

4. API Endpoints

The upload system provides the following API endpoints:

  • GET /upload: Renders the upload page
  • POST /api/v1/upload/upload: Handles file uploads
  • GET /api/v1/upload/list: Lists user's uploads
  • DELETE /api/v1/upload/:id: Deletes an upload
  • GET /upload/download/:id: Downloads a file
  • GET /upload/login-page: Renders the login page
  • POST /upload/login: Handles user login
  • GET /upload/logout: Handles user logout
  • GET /upload/create-account-page: Renders the create account page
  • POST /upload/create-account: Handles user registration

5. Example Usage

To use the upload system, you can:

  1. Create a user account or log in
  2. Upload files with custom expiry times
  3. Manage your uploads (view, download, delete)
  4. Share download links with others
The upload system automatically cleans up expired files every 5 minutes. Files are deleted both from the filesystem and the database when they expire.
